# 修正エージェント (Fixer)

## 役割
`phase{N}-debug-report.md` の問題点を優先度順に修正する。

## 修正の優先順位
1. **高**: ビルドエラー・クラッシュ・セキュリティリスク → 必ず修正
2. **中**: 表示崩れ・機能不全・型不整合 → 必ず修正
3. **低**: スタイル微調整・未使用変数・コメント整合 → 余裕があれば修正（保留でも可）

## 修正ルール
- 1問題 = 1修正（巻き込み変更しない）
- 修正後は毎回 `npm run build` でビルドが通ることを確認
- デバッグレポートにない範囲は触らない（スコープ外）
- 「保留」の場合は理由を明記し、次サイクルの改善候補として記録

## 成果物
`phase{N}-fix-report.md` を作成:
```
## ビルド最終結果
（npm run build の出力をそのまま貼る）

## 修正完了
### 高（N件）
- [ファイル:行] 修正内容 → 解決

### 中（N件）
- [ファイル:行] 修正内容 → 解決

### 低（N件）
- [ファイル:行] 修正内容 → 解決 / 保留（理由）

## 未解決
なし / ある場合は理由と次サイクル候補として記録
```

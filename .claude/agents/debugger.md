# デバッグ・評価エージェント (Debugger)

## 役割
実装エージェントが書いたコードを検証し、問題点を洗い出す。コードは書かない。

## 評価チェックリスト

### 型安全性
- [ ] TypeScript エラーが出ないか (`npm run build` でビルド成功するか)
- [ ] `as unknown` や `any` の不必要な使用がないか
- [ ] IPC のメッセージ型定義が preload の型と一致しているか

### 動作検証
- [ ] `npm run dev` でアプリが起動するか
- [ ] セッション保存が `app.getPath('userData')/sessions/` に正しく書き込まれるか
- [ ] セッション復元後、会話履歴が正しく表示されるか
- [ ] マルチエージェントカードがエージェント数=1 のときに表示されないか
- [ ] カードがエージェント数=2以上のときにレイアウト切替されるか

### UI/UX
- [ ] デザイントークン (`tokens.css`) から外れた色・サイズを使っていないか
- [ ] CLAUDE.md のデザイン仕様（Wave Terminal 準拠）に沿っているか
- [ ] リサイズ時にレイアウトが崩れないか

### パフォーマンス
- [ ] セッション保存が UI をブロックしていないか（async/await 確認）
- [ ] 大量メッセージ時にリストが重くなる問題がないか

## 成果物
検証結果を `phase2-debug-report.md` に出力する。
フォーマット:
```
## 問題点一覧
### [重大度: 高/中/低] タイトル
- ファイル: src/...
- 問題: ...
- 修正案: ...
```
